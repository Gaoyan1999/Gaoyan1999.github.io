            # Ngnix

            # 简介

            Nginx ：一个高性能的HTTP和反向代理的web服务器

            占用内存小，并发能力高，支持热部署

            基本概念

            • 反向代理
            • 负载均衡
            • 动静分离

            # 反向代理

            正向代理：用户想要访问某个网站，不直接访问，而是通过在客户端配置一个代理服务器来访问。

            反向代理：客户端不需要配置代理服务器，而是想请求发送给代理服务器，代理服务器通过请求转发到真实的服务器，由代理服务器返回响应。其暴露的是代理服务器，而不是真正的目标服务器。

            # 负载均衡

            随着并发量的增大，原来的请求响应架构已经无法满足日益增长的需求。我们可以增加服务器的数量，将请求分发到不同的服务器上。

            # 动静分离

            为了加快网站的解析速度，把静态页面和动态页面分开进行部署。Tomcat只部署servlet、jsp等动态资源，而又另外一个服务器专门加载静态页面。

            # 常用命令

            • nginx-v： 查看版本

            • nginx -s reload:重新加载

            • Nginx -s stop :关闭

            • service nginx start:启动

            # 配置文件

            nginx配置文件位置 /etc/nginx/nginx.conf

            主要分为三个部分 全局块、events、http

            全局块

            设置影响nginx服务器整体运行的配置命令，用户组，worker process、进程PID 存放路径、日志存放路径和其他配置文件的引入。

            events

            nginx服务器和用户网络的链接

            worker_connections 1024 ;支持最大连接数

            Http

            http全局配置的指令。其中包含全局http全局块和server块

            # 配置反向代理

            使用nginx跳转到不同的其他url

            在linux下配置tomcat 容器。使用nginx反向代理访问Tomcat

            • 在nginx的配置文件下配置需要跳转的url地址
                # Load configuration files for the default server block.
        include /etc/nginx/default.d/*.conf;

        location / {
        proxy_pass http://127.0.0.1:8080;

        }

            • 重新加载nginx

               nginx -s reload

            • 访问

              在本地直接访问服务器上的的80端口时，nginx会自动的去请求本地（指服务器的本地）所属Tomcat的8080端口，将相应结果返回给我们（请求者）。

            根据访问路径的不同跳转到不同端口的服务

            • 准备两个Tomcat容器，分别有不同的html页面

            • 在配置文件中加入反向代理

            • 开放对应的端口号
